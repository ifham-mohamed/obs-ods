# DB - Indexing

## What is Indexing?

**Indexing** is a data structure technique which allows you to quickly retrieve records from a database file. 

An Index is a small table having only two columns. 
1. The first column comprises a copy of the primary or candidate key of a table. 
2. Its second column contains a set of [pointers](https://www.guru99.com/c-pointers.html) for holding the address of the disk block where that specific key value stored.

## Primary Index in DBMS

Primary Index is an ordered file which is fixed length size with two fields. The first field is the same a primary key and second, filed is pointed to that specific data block. In the primary Index, there is always one to one relationship between the entries in the index table.

The primary Indexing in DBMS is also further divided into two types.

- Dense Index
- Sparse Index

### Dense Index

In a dense index, a record is created for every search key valued in the database. 

This helps you to search faster but needs more space to store index records. In this Indexing, method records contain search key value and points to the real record on the disk.

[![Dense Index](https://www.guru99.com/images/1/070119_0833_IndexinginD2.png)](https://www.guru99.com/images/1/070119_0833_IndexinginD2.png)

### Sparse Index

It is an index record that appears for only some of the values in the file. Sparse Index helps you to resolve the issues of dense Indexing in [DBMS](https://www.guru99.com/dbms-tutorial.html). In this method of indexing technique, a range of index columns stores the same data block address, and when data needs to be retrieved, the block address will be fetched.

However, sparse Index stores index records for only some search-key values. It needs less space, less maintenance overhead for insertion, and deletions but It is slower compared to the dense Index for locating records.

## Secondary Index in DBMS

The secondary Index in DBMS can be generated by a field which has a unique value for each record, and it should be a candidate key. It is also known as a non-clustering index.

This two-level database indexing technique is used to reduce the mapping size of the first level. For the first level, a large range of numbers is selected because of this; the mapping size always remains small.

### Secondary Index Example

Let’s understand secondary indexing with a database index example:

In a bank account database, data is stored sequentially by acc_no; you may want to find all accounts in of a specific branch of ABC bank.

Here, you can have a secondary index in DBMS for every search-key. Index record is a record point to a bucket that contains pointers to all the records with their specific search-key value.

## Clustering Index in DBMS

In a clustered index, records themselves are stored in the Index and not pointers. Sometimes the Index is created on non-primary key columns which might not be unique for each record. In such a situation, you can group two or more columns to get the unique values and create an index which is called clustered Index. This also helps you to identify the record faster.

**Example:**

Let’s assume that a company recruited many employees in various departments. In this case, clustering indexing in DBMS should be created for all employees who belong to the same dept.

It is considered in a single cluster, and index points point to the cluster as a whole. Here, Department _no is a non-unique key.

## B-Tree Index

B-tree index is the widely used data structures for tree based indexing in DBMS. It is a multilevel format of tree based indexing in DBMS technique which has balanced [binary search trees](https://www.guru99.com/binary-search-tree-data-structure.html). All leaf nodes of the B tree signify actual data pointers.

Moreover, all leaf nodes are interlinked with a link list, which allows a B tree to support both random and sequential access.

Indexing in databases is crucial for optimizing data retrieval and improving overall performance. Here’s a detailed explanation of the types of indexing in a Database Management System (DBMS), along with real-world examples and best practices.

## Types of Indexing in DBMS

### 1. **Primary Indexing**
Primary indexing involves creating an index based on the primary key of a table. This index is ordered and consists of two fields: the primary key and a pointer to the actual data block.

- **Types of Primary Indexing**:
  - **Dense Index**: Contains an index entry for every search key value. This allows for faster searches but consumes more storage.
  - **Sparse Index**: Contains index entries for only some of the search key values, which reduces storage but may slow down searches.

### 2. **Secondary Indexing**
Secondary indexing allows for indexing on non-primary key columns. It is also known as a non-clustering index. This type of index helps in searching records based on attributes that are not the primary key.

- **Example**: In a bank database, if accounts are stored by account number, a secondary index can be created for branch codes to quickly find all accounts in a specific branch.

### 3. **Clustering Index**
A clustered index stores the actual records in the index itself rather than pointers. This can be created on non-unique columns by combining multiple columns to form a unique key.

- **Example**: In a university database, if students are enrolled in various courses, a clustered index can be created on the course ID to group all students belonging to the same course.

### 4. **Multilevel Index**
Multilevel indexing is used when a primary index cannot fit into memory. It creates a hierarchy of indexes, allowing for reduced disk access.

- **Example**: For a large employee database, a first-level index can point to second-level indexes, which in turn point to the actual records.

### 5. **B-Tree Index**
B-Tree indexing is a balanced tree structure that allows for efficient searching, insertion, and deletion. All leaf nodes contain pointers to the actual data.

- **Example**: B-Trees are commonly used in file systems and databases to maintain sorted data and allow for efficient range queries.

### Explanation of Key Concepts

- **Types of Indexing**:
  - **Primary Indexing**: Ensures that each record can be accessed quickly using its primary key.
  - **Secondary Indexing**: Facilitates searches based on attributes other than the primary key, enhancing flexibility.
  - **Clustering Index**: Groups records based on a common attribute, improving access speed for related records.
  - **Dense and Sparse Indexing**: Dense indexing provides a complete index for all records, while sparse indexing saves space by indexing only some records.
  - **Multilevel Index**: Reduces the size of the index when dealing with large datasets, improving performance.
  - **B-Tree Index**: A versatile index structure that supports efficient searching and can handle dynamic data.

- **Types of File Organization**:
  - **Sequential File Organization**: Simple but inefficient for large datasets; records are stored in a specific order.
  - **Heap File Organization**: Records are stored randomly, making it easy to insert but inefficient for searching.
  - **Hash File Organization**: Uses hashing for fast access but can lead to data loss if collisions occur.

## Advantages of Indexing

- **Faster Data Retrieval**: Indexes significantly reduce the time required to search for records.
- **Reduced I/O Operations**: Fewer disk accesses are needed to retrieve data, improving performance.
- **Efficient Sorting**: Indexes can help maintain sorted order, facilitating faster range queries.

## Disadvantages of Indexing

- **Storage Overhead**: Indexes consume additional storage space, especially dense indexes.
- **Maintenance Cost**: Indexes need to be updated with every insert, delete, or update operation, which can slow down these operations.
- **Complexity**: Managing multiple indexes can increase the complexity of database administration.

## Real-World Examples

1. **E-commerce Websites**: Use secondary indexes to quickly retrieve product details based on category or price range, enhancing user experience during searches.

2. **Banking Systems**: Implement clustered indexes on account numbers to speed up transactions and queries related to specific accounts.

3. **Library Management Systems**: Use B-Tree indexing to allow efficient searching of books by title, author, or ISBN.

## Best Practices for Indexing

1. **Choose the Right Index Type**: Analyze the queries frequently executed against the database and choose the appropriate index type (primary, secondary, clustered) based on the access patterns.

2. **Limit the Number of Indexes**: Avoid creating too many indexes on a single table, as this can lead to increased maintenance costs and slower write operations.

3. **Regularly Monitor and Optimize**: Periodically review index usage and performance. Remove unused indexes and consider adding new ones based on changing query patterns.

4. **Use Composite Indexes Wisely**: When multiple columns are frequently queried together, consider creating composite indexes to improve performance.

5. **Test Performance Impact**: Before implementing indexing changes in production, test their impact on performance in a staging environment to ensure they provide the desired benefits.
